Hi, I'm Eric Gaby.

I work in security engineering with a focus on application security, web application penetration testing, systems security, and the practical work of helping teams build software with fewer sharp edges. I like security best when it is hands-on, explainable, and close enough to the code that fixes are more than a PDF disappearing into a ticket queue.

This site is a home for the things I keep coming back to:

- Building and operating CTF challenges
- Application security testing and secure SDLC work
- Threat modeling, vulnerability management, and defensive engineering
- Infrastructure projects that make technical teams faster and safer
- Notes from the long, ongoing process of learning in public

## What I Do

Most of my work lives at the intersection of software, infrastructure, and security. That includes reviewing application behavior, testing web apps for vulnerabilities, thinking through architecture risk, writing reports that engineers can actually act on, and building tooling or automation where it saves repeated pain.

I also spend a lot of time around CTFs. I enjoy the craft of designing challenges that are fair, weird, and educational: the kind where a solver has to notice one small truth about a system and then pull the thread. The BSides Fort Wayne CTF work in my repos is part security lab, part puzzle design, and part event operations.

## Background

I earned a Bachelor of Science in Computer Science and Information Technology and an MBA from Trine University. Trine's cybersecurity coursework gave me the kind of hands-on foundation that made the field feel immediate: web security, systems, software engineering, and the habit of learning fast when the problem is unfamiliar.

Outside day-to-day security work, I have stayed involved in technical education and community work, including advisory-board service and CTF volunteering. I care a lot about making security approachable without sanding off the hard parts.

## Current Interests

- Making CTF challenges more resistant to copy-paste solving while still being generous to curious beginners
- Secure software lifecycle practices that developers can realistically adopt
- Appsec patterns in Python, Flask, .NET, and cloud-hosted internal apps
- Badge firmware, embedded puzzles, and hardware-adjacent security toys
- Better ways to teach offensive techniques without glamorizing fragile systems
